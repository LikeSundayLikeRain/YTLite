<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Upstream .deb download | Untrusted binary downloaded from GitHub releases into CI runner |
| Binary patch write | Writing raw bytes to a specific offset in a downloaded binary |
| Patched .deb consumed by cyan | Modified .deb injected into YouTube IPA for end-user installation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Upstream .deb | mitigate | SHA256 verification (D-07): hard-coded hash `cc48b87d...` checked via `shasum -a 256` before any extraction. CI fails on mismatch. |
| T-01-02 | Tampering | Patch write | mitigate | Post-patch byte verification (D-05): `xxd -s $OFFSET -l 8 -p` compared against expected `00008052c0035fd6`. CI fails if bytes don't match. |
| T-01-03 | Information Disclosure | CI logs | accept | Before/after hex dumps (D-06) show 8 bytes of a public binary. No secrets or PII exposed. Hex dump is intentional for auditability. |
| T-01-04 | Denial of Service | SHA256 mismatch on upstream change | accept | If upstream releases a different v5.2 binary, CI correctly fails. This is intentional -- we only patch a known binary. Future versions are out of scope (REQUIREMENTS.md). |
| T-01-05 | Elevation of Privilege | Patched dylib behavior | accept | The patch changes dvnLocked to return 0 (unlocked) -- this is the explicit project goal. The dylib runs in the YouTube app sandbox with no additional privileges beyond what the unpatched tweak already has. |
| T-01-06 | Repudiation | Patch audit trail | mitigate | Before/after hex dumps logged in CI output (D-06). SHA256 of input .deb logged. All values are deterministic and reproducible. |
</threat_model>
