Create the directories `.planning/phases/02-verification/scripts/` and `.planning/phases/02-verification/evidence/`.

Create `verify_patch.py` -- a Python 3 script using capstone (version 5.0.7, already installed) that:

1. Downloads the upstream v5.2 .deb from `https://github.com/dayanch96/YTLite/releases/download/v5.2/com.dvntm.ytlite_5.2_iphoneos-arm.deb`
2. Verifies SHA256 matches `cc48b87d76605b26e75414e2bc3d678b5f32f3c7bc57710754f29dc33b2fff9b`
3. Extracts the .deb using `subprocess.run(["dpkg-deb", "-R", deb_path, extract_dir])`
4. Locates dylib at `{extract_dir}/Library/MobileSubstrate/DynamicLibraries/YTLite.dylib`
5. Reads 8 bytes at offset `0x1eb64` (the ORIGINAL unpatched bytes)
6. Logs the original bytes hex
7. Simulates the patch: writes `\x00\x00\x80\x52\xc0\x03\x5f\xd6` at offset `0x1eb64` (to a copy, not modifying downloaded file)
8. Reads 8 bytes at offset `0x1eb64` from the patched copy
9. Asserts patched bytes equal `00008052c0035fd6`
10. Uses `capstone.Cs(CS_ARCH_ARM64, CS_MODE_ARM)` to disassemble the 8 patched bytes at address `0x1eb64`
11. Asserts first instruction is `mov` with operand containing `w0` and `#0`
12. Asserts second instruction is `ret`
13. Also disassembles the function at offset `0x1eb78` (dvnCheck) to document what it does (per D-08, we do NOT patch it, but we record its instructions for reference if D-09 triggers later)
14. Prints structured output: PASS/FAIL for each check, full disassembly listing

Also verify using `otool` as cross-reference:
- Run `otool -tv {patched_dylib}` piped through grep for the address range around `1eb64`
- Capture output for the report

Run the script. Capture all stdout/stderr.
